Atividade Prática 2 – Simulação de Entrevista
  Roteiro com Anotações (Resumo da Entrevista)
Entrevistado: Maria (Coordenadora)
       Usa planilhas Excel para tudo (controle de alunos, notas, disciplinas) 
Trabalha há 15 anos como coordenadora 
Relatou retrabalho frequente ao atualizar dados em várias planilhas 
Disse que: 
às vezes perde dados ou versões 
é difícil garantir que tudo está atualizado
  Problemas identificados:
•	Falta de integração 
•	Alto risco de erro manual 
•	Dificuldade para gerar relatórios 
•	Tempo alto gasto com tarefas operacionais 
 
 Necessidades mencionadas:
•	Sistema único integrado 
•	Relatórios automáticos 
•	Controle de desempenho dos alunos 
•	Facilidade de uso (não quer algo complexo) 
________________________________________
         Lista de Requisitos Identificados
       RF (Requisitos Funcionais)
•	RF01: Sistema deve permitir cadastro e gestão de alunos 
•	RF02: Sistema deve permitir controle de disciplinas e turmas 
•	RF03: Sistema deve gerar relatórios de desempenho acadêmico 
•	RF04: Sistema deve permitir atualização centralizada dos dados 
•	RF05: Sistema deve permitir consulta rápida de informações 
________________________________________
       RNF (Requisitos Não Funcionais)
•	RNF01: Sistema deve ser fácil de usar (usabilidade alta) 
•	RNF02: Tempo de resposta inferior a 2 segundos 
•	RNF03: Sistema deve garantir segurança e integridade dos dados 
•	RNF04: Disponibilidade mínima de 99% 
•	RNF05: Interface deve ser intuitiva (baixo esforço de aprendizado) 
________________________________________
      RN (Regras de Negócio)
•	RN01: Cada aluno deve estar vinculado a um curso 
•	RN02: Notas só podem ser alteradas por usuários autorizados 
•	RN03: Relatórios devem considerar período letivo 
•	RN04: Dados acadêmicos devem ser mantidos históricos (não podem ser apagados)
